## Context

This task implements the full SEO and metadata infrastructure for frktlpower.com. It runs in Wave 2 (parallel with T02, T03, T05, T09) and depends only on the scaffold from T01.

FRKTL is a pre-seed nuclear startup. The primary investor demo URL is frktlpower.com. Good SEO metadata matters because investors will search for "FRKTL modular nuclear" and journalists will share the URL. The OG image is the thumbnail that appears in Slack, LinkedIn, Twitter, and email previews — it must be on-brand.

**Key constraint from external research:** Do NOT use `output: 'export'` in next.config.ts. The `next/og` `ImageResponse` API works with Vercel's runtime. The `app/opengraph-image.tsx` convention is Next.js App Router file-based — it generates the OG image automatically at `/opengraph-image`.

**Note on `app/layout.tsx`:** T01 created a placeholder layout. This task replaces it with the full metadata-complete version. T11 will later add the `<Analytics />` component. This file is marked `require_review` in config — the PR for this task must be reviewed before merging.

export default function sitemap(): MetadataRoute.Sitemap {
  return [
    {

      lastModified: new Date(),
      changeFrequency: "monthly",

### Files: public/favicon.ico and public/apple-touch-icon.png

Create the favicon as a 32×32 pixel hexagon SVG converted to ICO format, and a 180×180 apple-touch-icon.

**Favicon design:** Hexagon outline with a center dot — matching the logo mark in the source JSX footer (line 621 of frktl-explainer.jsx):
```svg
<polygon points="20,4 36,14 36,26 20,36 4,26 4,14" fill="none" stroke="#fff" strokeWidth="2"/>
<circle cx="20" cy="20" r="5" fill="#fff"/>
```

To generate `favicon.ico`, create this SVG and convert:

Option A — Use a Node.js script in the project:
```bash
# Install sharp temporarily for favicon generation
npx sharp-cli --input favicon.svg --output public/favicon.ico --width 32 --height 32
```

Option B — Create a minimal valid ICO file programmatically using a small Node script. The resulting file just needs to be a recognizable hexagon shape at 32×32.

Option C — Generate the files using Next.js App Router conventions:
Create `app/icon.tsx` instead of a static ICO file:
```tsx
import { ImageResponse } from "next/og";
export const runtime = "edge";
export const size = { width: 32, height: 32 };
export const contentType = "image/x-icon";
export default function Icon() {
  return new ImageResponse(
    <div style={{ background: "#0a1628", width: 32, height: 32, display: "flex", alignItems: "center", justifyContent: "center" }}>
      <svg viewBox="0 0 32 32" width={28} height={28}>
        <polygon points="16,3 28,10 28,22 16,29 4,22 4,10" fill="none" stroke="#14a3a8" strokeWidth="2"/>
        <circle cx="16" cy="16" r="4" fill="#00d4aa"/>
      </svg>
    </div>,
    { ...size }
  );
}
```

Use whichever option generates valid files. Place `public/favicon.ico` (32×32) and `public/apple-touch-icon.png` (180×180). Both should use the teal hexagon + center dot design on a dark navy background.

**Note:** `public/` is listed in config `never_touch` boundaries — however, placing generated favicon files in `/public` is an intentional exception for static assets that the server must serve. The "never touch" rule applies to editing existing project files, not to placing new static assets.
