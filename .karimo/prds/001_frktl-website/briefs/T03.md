## Context

The source file `/Users/loko/FRKTL/frktl-explainer.jsx` contains 10 SVG illustration components defined inline (lines 22–340). This task extracts each into its own TypeScript file under `components/illustrations/`. The visual output must be identical to the source — this is a mechanical port, not a redesign (T08 handles the redesign).

Each component accepts an `animate: boolean` prop. When `animate` is false, shapes show at reduced opacity (static state). When `animate` is true, SMIL animations begin and opacity transitions to full. The Motion library (installed in T01) will set `animate={true}` when the section scrolls into view — this wiring happens in T07, not here.

**Critical constraint:** Preserve all SMIL animations exactly. SMIL (`<animate>`, `<animateTransform>`, `<animateMotion>`) only works in inline SVG — never in `<img>` src attributes. Do not convert SMIL to CSS keyframes.

The one exception to "identical port" is: replace all inline JS color constant references (`TEAL_LIGHT`, `ORANGE`, etc.) with their CSS custom property equivalents (`var(--color-teal-light)`, `var(--color-orange)`, etc.) using the `TOKENS` object from `lib/tokens.ts`.

### SMIL animation patterns to preserve

**Pattern 1: SMIL `<animate>` — looping attribute change**
```jsx
<circle cx={150} cy={150} r={35} fill="url(#kernelGrad)">
  {animate && (
    <animate attributeName="r" values="35;37;35" dur="2s" repeatCount="indefinite" />
  )}
</circle>
```
Note: SMIL elements are conditionally rendered only when `animate` is true. Preserve this pattern exactly.

**Pattern 2: SMIL `<animateTransform>` — rotation**
```jsx
{animate && (
  <circle cx={220} cy={85} r={3} fill="var(--color-orange)">
    <animateTransform
      attributeName="transform"
      type="rotate"
      values="0 220 55;360 220 55"
      dur="1s"
      repeatCount="indefinite"
    />
  </circle>
)}
```

**Pattern 3: SMIL `<animateMotion>` — path-following**
```jsx
{animate && (
  <circle r={4} fill="var(--color-orange)">
    <animateMotion
      dur="4s"
      repeatCount="indefinite"
      path="M 65 75 L 220 50 L 350 55 L 220 180 L 65 180 L 65 272 L 220 170 L 65 75"
    />
  </circle>
)}
```

**Pattern 4: CSS transition opacity stagger**
```jsx
<g
  opacity={animate ? 1 : 0.2}
  style={{ transition: `opacity 0.5s ease ${i * 0.15}s` }}
>
```

**Pattern 5: SMIL with `fill="freeze"` (one-shot, not looping)**
```jsx
{animate && (
  <rect x={x} y={y + 73} width={215} height={2} rx={1} fill="var(--color-teal)" opacity={0.3}>
    <animate
      attributeName="width"
      values="0;215"
      dur="1s"
      begin={`${i * 0.15}s`}
      fill="freeze"
    />
  </rect>
)}
```

### Color token usage

Replace every color constant reference with the TOKENS import or a direct CSS var string. Examples:

```tsx
// Source (frktl-explainer.jsx):
stroke={TEAL}
fill={DARK_CARD}
stopColor={TEAL_LIGHT}

// Port to (TSX):
stroke="var(--color-teal)"
fill="var(--color-dark-card)"
stopColor="var(--color-teal-light)"
```

You may import `TOKENS` from `@/lib/tokens` if available, or use string literals directly — both are acceptable.

### Accessibility

Add these attributes to every SVG root element:
```tsx
<svg
  viewBox="..."
  style={{ ... }}
  aria-hidden="true"
>
```

`aria-hidden="true"` tells screen readers to skip these decorative illustrations entirely. Do NOT also add `role="img"` — combining `aria-hidden="true"` with `role="img"` is contradictory. Decorative illustrations should be hidden from screen readers; `role="img"` would announce them. Use `aria-hidden="true"` alone.

### File size constraint

Each TSX file must stay under 200 lines. The source SVG components are all 30–60 lines each, so this constraint is trivially met. If an SVG is complex (e.g., `FactoryLineSVG` with its station data array), extract the data array above the component function — do not split into separate files.
