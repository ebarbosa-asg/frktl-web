## Context

This is the first task and the only Wave 1 task. Every other task (T02–T11) depends on it. The goal is to produce a working, properly configured Next.js 15 App Router project at `/Users/loko/frktl-web/` with Tailwind CSS v4, TypeScript strict mode, all 16 CSS color tokens declared as custom properties, and the complete directory structure the later tasks will populate.

The project is `frktl-web` — the public website for FRKTL Energy (frktlpower.com), a modular nuclear reactor startup. The site is a single-page scroll explainer ported from a 627-line React JSX component. This task establishes the foundation: no content, no components — just a working scaffold that builds and type-checks cleanly.

**Important note from external research:** Do NOT use `output: 'export'` in next.config.ts. Static export disables `next/image`, `next/og`, and Vercel Analytics. The site should deploy to Vercel without static export. The site will still be fully static because there is no server-side data fetching.

## Technical Specification

### Step 1 — Initialize the project

Run this exact command inside `/Users/loko/frktl-web/` (the directory already exists as a git repo):

```bash
cd /Users/loko/frktl-web
npx create-next-app@latest . --ts --app --no-tailwind --src-dir=false --import-alias="@/*" --yes
```

Flags explained:
- `--ts` — TypeScript
- `--app` — App Router (not Pages Router)
- `--no-tailwind` — we install Tailwind v4 manually (create-next-app would install v3)
- `--src-dir=false` — no src/ wrapper, files live at root
- `--import-alias="@/*"` — `@/` maps to project root

### Step 2 — Install Tailwind CSS v4

Tailwind v4 uses a CSS-first configuration — no tailwind.config.js needed. Install:

```bash
npm install tailwindcss @tailwindcss/postcss
```

Create `postcss.config.mjs`:
```js
const config = {
  plugins: {
    "@tailwindcss/postcss": {},
  },
};
export default config;
```

### Step 3 — Install additional packages

Ensure `strict: true` is set and path aliases are correct:

```json

    "skipLibCheck": true,
```

### Step 6 — Configure app/globals.css

Replace the default content entirely with:

```css
@import "tailwindcss";

:root {
  /* FRKTL Brand Palette — ported from frktl-explainer.jsx constants */
  --color-teal: #0d7377;
  --color-teal-light: #14a3a8;
  --color-teal-dark: #0a5c5f;
  --color-cyan: #00d4aa;
  --color-dark: #0a1628;
  --color-dark-mid: #0f2035;
  --color-dark-card: #111d30;
  --color-slate: #1a2a42;
  --color-text: #c8dce8;
  --color-text-bright: #e8f4f8;
  --color-orange: #ff6b35;
  --color-gold: #fbbf24;
  --color-cream: #fff8ee;
  --color-steel: #8899aa;
  --color-purple: #a78bfa;
  --color-green: #34d399;
  --color-red-soft: #f87171;

  /* 50s Atomic-Age additional tokens (for T08 visual redesign) */
  --color-atomic-orange: #e8521a;
  --color-atomic-cream: #f5e6c8;
  --color-atomic-rust: #8b3a1a;
}

* {
  box-sizing: border-box;
  margin: 0;
  padding: 0;
}

html {
  scroll-behavior: smooth;
}

body {
  background: var(--color-dark);
  color: var(--color-text);
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
}
```

**Critical:** Tailwind v4 uses `@import "tailwindcss"` (not `@tailwind base; @tailwind components; @tailwind utilities`). The CSS custom properties declared on `:root` are automatically available as Tailwind theme tokens via the `--color-*` naming convention.
