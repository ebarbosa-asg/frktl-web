# Brief: T08 — Visual Redesign: 50s Atomic-Age Layer

**Wave:** 3 | **Complexity:** 5/10 | **Priority:** Must | **Depends on:** T02, T03, T05

---

## Context

This is the centerpiece design task. The existing JSX produces a competent dark-navy explainer with thin SVG diagrams. This task transforms it into the "Nuclear Optimism" visual language — a fusion of 50s atomic-age editorial, cute 3D isometric (partially handled by T09), and modern tech-startup grid.

The task has three sub-scopes:
- **8a Typography:** Atomic Age font for chapter labels, large hero headline, stats strip upgrades
- **8b Decorative system:** AtomicStarburst component, halftone overlay, blob borders, danger tape
- **8c SVG illustration redesign:** All 10 SVG files with 3–4px stroke, round linecaps, warm fills

**All SMIL animations must be preserved.** This task only changes visual style properties (strokeWidth, colors, fills, linecaps) — not animation logic.

## Design Direction

### The Three Layers in This Task

**Layer 1 — 50s Atomic-Age Editorial (primary for T08):**
- Atomic Age Google Font on all 17 section TAG labels
- Bold orange/rust chapter marker numbers
- Section backgrounds alternating: dark (`#0a1628`) and dark-mid (`#0f2035`)
- Atomic starburst SVG decorations at section transitions
- Halftone dot overlay (CSS radial-gradient) on alternating dark sections
- "DANGER tape" diagonal stripe on the safety section

**Layer 2 — Modern Grid (typography/layout):**
- Hero headline at `clamp(48px, 8vw, 100px)` on desktop
- Stats strip numerals at 64px+ with DM Sans labels below
- Learning curve cards with organic amoeba/blob border-radius (using CSS `border-radius` multi-value syntax)

**Layer 3 — Bold SVG Illustrations:**
- All 10 SVG components: stroke-width 3–4px, `strokeLinecap="round"`, `strokeLinejoin="round"`
- Key elements get warm fills: TRISO kernel gets gold/orange, coolant channels get teal, fuel gets gold
- Preserved: all SMIL animation logic, all viewBox dimensions, all coordinate positions

### Reference Colors for This Task

### Sub-scope 8b: Decorative System

#### File: components/ui/AtomicStarburst.tsx

```tsx
interface AtomicStarburstProps {
  points?: number;        // Default 8
  innerRadius?: number;   // Default 40
  outerRadius?: number;   // Default 60
  color?: string;         // Default var(--color-orange)
  opacity?: number;       // Default 0.15
  size?: number;          // SVG size in px, default 120
  className?: string;
}

export function AtomicStarburst({
  points = 8,
  innerRadius = 40,
  outerRadius = 60,
  color = "var(--color-orange)",
  opacity = 0.15,
  size = 120,
  className,
}: AtomicStarburstProps) {
  const cx = 100;
  const cy = 100;
  const viewSize = 200;

  // Generate starburst points
  const pathPoints: [number, number][] = [];
  for (let i = 0; i < points * 2; i++) {
    const angle = (Math.PI / points) * i - Math.PI / 2;
    const r = i % 2 === 0 ? outerRadius : innerRadius;
    pathPoints.push([cx + r * Math.cos(angle), cy + r * Math.sin(angle)]);
  }

  const d = pathPoints
    .map((p, i) => `${i === 0 ? "M" : "L"} ${p[0].toFixed(1)} ${p[1].toFixed(1)}`)
    .join(" ") + " Z";

  return (
    <svg
      viewBox={`0 0 ${viewSize} ${viewSize}`}
      width={size}
      height={size}
      className={className}
      aria-hidden="true"
    >
      <path d={d} fill={color} opacity={opacity} />
    </svg>
  );
}
```

**Usage in sections:** Place `<AtomicStarburst>` as absolutely positioned decorative elements behind section headings or at section borders. Used in at least 3 section backgrounds.

Example placement:
```tsx
<div className="relative">
  <AtomicStarburst
    points={12}
    color="var(--color-orange)"
    opacity={0.08}
    size={200}
    className="absolute -top-4 -right-4 pointer-events-none"
  />
  {/* Section content */}
</div>
```

### Sub-scope 8c: SVG Illustration Redesign

Edit all 10 files in `components/illustrations/`. For each file, apply these changes systematically:

**Universal changes:**
1. All `strokeWidth` values: set to 3 (minimum), or 3.5–4 for primary shapes
2. Add `strokeLinecap="round"` and `strokeLinejoin="round"` to all `<path>`, `<line>`, `<polyline>`, and `<polygon>` elements
3. Add warm fill colors to key elements (see below)
4. Do NOT change any coordinate values, viewBox, animation attribute values, or SMIL elements

**Per-component warm fill additions:**

`TrisoParticleSVG.tsx`:
- The UCO kernel gradient: change outer stop from `#b8860b` to `var(--color-gold)`, inner stop from `#ffd700` to `var(--color-orange)`
- The SiC layer (sicGrad): use `var(--color-teal)` to `var(--color-teal-dark)` (already uses TOKENS)
- Increase outer PyC and buffer circle opacity when not animated from 0.3 → 0.5 for better visibility

`PrismaticBlockSVG.tsx`:
- The hexagonal block: increase `strokeWidth` on the polygon from 2 to 3.5
- Fuel compact circles (gold dots): increase `r` from 5 to 6 when animate=true
- Coolant channel circles: add a subtle fill `fill="var(--color-dark)"` to make them more distinct

`ReactorCoreSVG.tsx`:
