## Part 3: Custom Domain (frktlpower.com)

### Step 8 — Add domain in Vercel dashboard

1. In the Vercel project dashboard, go to Settings → Domains
2. Add `frktlpower.com`
3. Add `www.frktlpower.com`
4. Vercel will show you the DNS records to add

### Step 9 — DNS configuration at your registrar

Vercel provides two options. Use **Option A** (A record) if the registrar supports CNAME flattening at the apex; otherwise use **Option B** (A record for apex + CNAME for www).

**Option A — Preferred (Vercel nameservers, if available):**
If Eduardo's registrar allows custom nameservers, point the domain to Vercel's nameservers:
```
ns1.vercel-dns.com
ns2.vercel-dns.com
```
Vercel then manages all DNS automatically.

**Option B — Individual DNS records:**

For the apex domain (`frktlpower.com`):
```
Type: A
Name: @ (or blank, depending on registrar)
Value: 76.76.21.21
TTL: 3600 (or Auto)
```

For `www.frktlpower.com`:
```
Type: CNAME
Name: www
Value: cname.vercel-dns.com
TTL: 3600 (or Auto)
```

**Note for Eduardo:** DNS propagation takes 5 minutes to 24 hours, depending on your registrar's TTL. Most registrars propagate within 30 minutes. You can check propagation status at dnschecker.org.

**Common registrars — where to find DNS settings:**
- Namecheap: Domain List → Manage → Advanced DNS
- GoDaddy: My Products → Domain → DNS Management  
- Cloudflare: DNS tab in the domain dashboard (if using Cloudflare already — use Proxy status = DNS Only for the A record)
- Google Domains / Squarespace Domains: DNS → Custom Records

### Step 10 — Verify HTTPS

After DNS propagates, Vercel automatically provisions a Let's Encrypt TLS certificate. Verify:
- `https://frktlpower.com` loads the site with green padlock
- `https://www.frktlpower.com` redirects to `https://frktlpower.com` (Vercel handles the www redirect)

## Part 4: Verification Checklist

### Step 11 — Core Web Vitals check

1. Go to Vercel project dashboard → Speed Insights
2. Wait for real user data (or use Vercel's automatic synthetic test)
3. Target: LCP < 2.5s on the production URL

If LCP is above 2.5s, common culprits:
- Fonts not loading from `next/font/google` (check layout.tsx)
- Spline scene blocking render (verify it's lazy-loaded with `ssr: false`)
- Large JS bundle — run `npm run build` and check the bundle size output

### Step 12 — OG image verification

Check the OG image via an external checker:
- opengraph.xyz: enter `https://frktlpower.com`
- Or share the URL in Slack/iMessage and verify the preview shows the correct 1200×630 image

### Step 13 — robots.txt and sitemap.xml

```bash
curl https://frktlpower.com/robots.txt
# Expected output:
# User-agent: *
# Allow: /
# Sitemap: https://frktlpower.com/sitemap.xml

curl https://frktlpower.com/sitemap.xml
# Expected output: XML sitemap with https://frktlpower.com as the first URL
```

### Step 14 — Enable Vercel Speed Insights (if not automatic)

In Vercel project dashboard → Analytics tab → enable "Speed Insights." This is separate from the `<Analytics />` component (which tracks page views) — Speed Insights tracks Core Web Vitals from real users.
